Comment choisir le bon port SMTP (port 25, 587, 465 ou 2525)

Matteo Duò, 22 août 2023

Vous avez du mal à trouver le bon port SMTP à utiliser ?

Si vous utilisez un client de messagerie comme Apple Mail ou Outlook pour envoyer des emails, ce client de messagerie utilise probablement aussi le protocole SMTP pour téléverser vos emails sortants sur votre serveur de messagerie (bien que les clients utilisent généralement d’autres protocoles comme IMAP ou POP3 pour téléverser les emails entrants sur l’application).

En outre, si vous êtes de ceux qui se débattent avec la délivrabilité des emails sur WordPress, l’un des meilleurs moyens de résoudre le problème est d’utiliser un service d’envoi SMTP comme SendGrid, ou Google Workspace.

Mais si vous essayez de configurer le SMTP avec votre client de messagerie ou votre site web WordPress, vous vous poserez probablement la question de savoir quel port SMTP utiliser.

Pour vous aider à choisir le bon port SMTP en fonction de vos besoins, nous allons examiner dans cet article tout ce qui concerne le port SMTP.

Qu'est-ce qu'un port SMTP ?

Un port SMTP est un point de terminaison de la communication qui gère les transferts d’informations d’un serveur à un autre. Alors que le protocole SMTP (Simple Mail Transfer Protocol) garantit que la plupart des e-mails sont envoyés sur le web, un port assure que les bonnes données d’e-mail sont envoyées au bon endroit.

Support

Comprenons ce processus plus en détail.

Comme nous l’avons dit, SMTP est le protocole standard de transmission des e-mails sur le web. C’est ce que les serveurs de messagerie utilisent pour envoyer et recevoir des e-mails sur Internet.

Par exemple, lorsque vous envoyez un e-mail, votre client de messagerie a besoin d’un moyen de téléverser l’e-mail sur le serveur de courrier sortant. Ensuite, le serveur de courrier sortant a besoin d’un moyen pour transférer votre courrier électronique au serveur de courrier entrant du destinataire.

Les serveurs de courrier électronique ressemblent beaucoup aux serveurs de sites web en ce sens que, même s’il peut y avoir une interface de nom de domaine conviviale la communication proprement dite se fait par l’intermédiaire d’adresses IP, comme 222.501.285.45 (pour en savoir plus sur la manière dont cela se passe, consultez notre introduction au système des noms de domaine (DNS).

Un « port » est un autre moyen d’aider les ordinateurs (comme deux serveurs de messagerie) à communiquer entre eux :

  • Une adresse IP permet d’identifier un ordinateur.
  • Un port identifie une application/service spécifique s’exécutant sur cet ordinateur, comme le SMTP.

Voici une analogie pour rendre le tout un peu plus humain :

Une adresse IP est l’adresse physique d’un complexe commercial. Un port est le numéro d’une entreprise spécifique dans ce complexe commercial.

Si vous voulez livrer quelque chose à cette entreprise, vous ne pouvez pas seulement l’adresser au complexe commercial, vous devez aussi vous assurer qu’il arrive au bon endroit à l’intérieur du complexe commercial.

L’IANA, l’organisation responsable de l’attribution des adresses IP mondiales et d’autres tâches, est également chargée d’enregistrer les numéros de port pour les services Internet communs, dont le SMTP.

Pourquoi votre port SMTP est-il important ?

Si vous souhaitez vous connecter à un serveur SMTP (comme le serveur SMTP Gmail), vous devez saisir à la fois son adresse IP et son numéro de port.

Cependant, il existe de nombreux ports SMTP communs (voir plus loin) et ils ne fonctionnent pas tous dans toutes les situations.

Par exemple, le port 25, le port SMTP standard pour le transfert des messages entre les serveurs de messagerie, est souvent bloqué par les FAI et les fournisseurs d’accès au cloud (y compris Google Cloud Platform, que Kinsta utilise).

Ainsi, si vous essayez de vous connecter à un serveur SMTP via le port 25, vous rencontrerez souvent des problèmes car de nombreux services bloquent le port 25.

Des ports différents pour des objectifs différents

Au-delà des implications ci-dessus, les différents ports SMTP ont également des objectifs différents.

La transmission SMTP comporte deux grandes étapes :

  • Soumission – soumission d’un message électronique à un serveur de courrier sortant. Par exemple, lorsque vous envoyez un e-mail dans Apple Mail, ce message doit être soumis au serveur de courrier sortant.
  • Relais – le processus de transmission du message entre deux serveurs. Ainsi, après qu’un e-mail a été « soumis » au serveur de courrier sortant, celui-ci « relaie » ce message au serveur de courrier du destinataire.

Si vous configurez votre client de messagerie ou votre site WordPress, vous êtes surtout concerné par la partie « soumission » du processus.

Bien que le « relais » soit certainement une partie importante du SMTP, la plupart des gens n’ont pas besoin de configurer leur propre serveur de courrier électronique.

Quel est le port utilisé par SMTP ?

Sur le web moderne, il n’y a pas un seul port SMTP. Au lieu de cela, il existe quatre ports SMTP communs :

  • 25
  • 587
  • 465
  • 2525

Passons-les en revue.

À quoi sert le port 25 ?

Le port 25 a été établi dès 1982, ce qui en fait le plus ancien port SMTP.

Le port 25 est toujours connu comme le port SMTP standard et il est surtout utilisé pour le relais SMTP.

Toutefois, si vous configurez votre site WordPress ou votre client de messagerie avec le protocole SMTP, généralement vous ne voulez pas utiliser le port 25 car la plupart des FAI résidentiels et des hébergeurs du cloud bloquent le port 25.

Pourquoi ? Parce que le port 25 est couramment utilisé pour envoyer du spam à partir d’ordinateurs compromis.

N’oubliez pas : il y a une différence entre la soumission SMTP et le relais. Ainsi, si le port 25 est parfait pour le relais SMTP, il n’est pas une bonne option pour la soumission SMTP.

À quoi sert le port 587 ?

Le port 587 est le port par défaut pour la soumission SMTP sur le web moderne. Bien que vous puissiez utiliser d’autres ports pour la soumission (plus d’informations plus loin), vous devez toujours commencer par le port 587 par défaut et n’utiliser un port différent que si les circonstances l’exigent (comme votre hôte qui bloque le port 587 pour une raison quelconque).

Le port 587 prend également en charge TLS, ce qui signifie que vous pouvez envoyer du courrier en toute sécurité.

À quoi sert le port 465 ?

Le port 465 était à l’origine enregistré pour le SMTPS (SMTP over SSL). Après un bref passage dans cette fonction, le port 465 a été réaffecté à un autre usage et déprécié.

Malgré cela, de nombreux FAI et fournisseurs d’hébergement du cloud supportent toujours le port 465 pour la soumission SMTP.

À quoi sert le port 2525 ?

Le port 2525 n’est pas un port SMTP officiel (tel que reconnu par l’IETF ou l’IANA). Toutefois, il est encore couramment utilisé comme alternative au port 587 pour la soumission SMTP et la plupart des FAI et des fournisseurs d’hébergement du cloud prennent en charge le port 2525 pour le SMTP.

Si le port 587 est bloqué, le port 2525 constitue une bonne alternative.

Quel port SMTP devez-vous utiliser ?

Nous avons abordé ce point ci-dessus, mais récapitulons comment choisir le bon port SMTP, car il est important de prendre la bonne décision.

Si vous configurez votre site WordPress ou votre client de messagerie pour envoyer des e-mails par SMTP (soumission), vous voudrez presque toujours utiliser le port 587. Là encore, il s’agit du port SMTP par défaut pour la soumission et il prend en charge la transmission sécurisée via TLS.

Si le port 587 est bloqué pour une raison quelconque, le port 2525 est une alternative courante. Là encore, il ne s’agit pas d’un port SMTP officiellement reconnu, mais il est couramment utilisé et soutenu par la plupart des fournisseurs.

Bien que de nombreux fournisseurs prennent encore en charge le port 465 pour le SMTP, ce n’est plus une norme acceptée et vous devriez toujours essayer d’utiliser les ports 587 et 2525 avant d’utiliser le port 465.

Enfin, si le port 25 est couramment utilisé pour le relais SMTP, vous ne devez pas l’utiliser lors de la configuration d’un client de messagerie ou d’un site web WordPress car la plupart des FAI et des hébergeurs du cloud bloquent le port 25.

Quels ports SMTP pouvez-vous utiliser chez Kinsta ?

Chez Kinsta, vous pouvez utiliser les ports :

  • 587
  • 465
  • 2525

Vous ne pouvez pas utiliser le port 25 car Kinsta utilise l’infrastructure de la plateforme Google Cloud et la plateforme Google Cloud bloque le port 25.

Résumé

Le SMTP joue un rôle important dans la circulation des emails sur Internet.

Pour améliorer la délivrabilité de votre site WordPress les e-mails transactionnels, vous pouvez configurer votre site WordPress pour envoyer des e-mails via SMTP. De plus, si vous utilisez un client de messagerie comme Apple Mail ou Outlook, votre client de messagerie utilise le SMTP pour soumettre les e-mails sortants au serveur de messagerie.

Afin de connecter votre site WordPress ou votre client de messagerie au serveur SMTP, vous devez saisir un port SMTP spécifique.

Il existe quatre ports SMTP communs :

  • 25
  • 587
  • 465
  • 2525

Le port 25 est couramment utilisé pour le relais SMTP, mais vous ne devez pas l’utiliser pour la soumission SMTP car la plupart des fournisseurs le bloquent.

Si vous souhaitez configurer votre site WordPress ou votre client de messagerie pour utiliser le SMTP, vous devez commencer par le port 587 comme premier choix, car c’est le port standard pour la soumission SMTP.

Et si le port 587 ne fonctionne pas, vous pouvez essayer le port 2525. Bien que ce ne soit pas un port SMTP officiellement reconnu, il est largement soutenu et il supporte le TLS pour une transmission sécurisée.
